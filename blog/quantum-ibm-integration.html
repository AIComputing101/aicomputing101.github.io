Quantum-101 Now Runs on Real IBM Quantum Hardware

Today, we’re excited to announce a game-changing update to quantum-computing-101: full integration with IBM Quantum hardware. This bridges our repository’s focus on from-basics-to-advanced learning with real-world quantum computing — no more limiting yourself to simulators.

What’s New in the Repository?

We’ve expanded the repo to include three key modules that build on our existing quantum foundations:

# New structure in quantum-computing-101
examples/
├── 01_basic_circuits/       # Existing: Simulator-focused
├── 02_ibm_quantum/          # New: Hardware-specific implementations
│   ├── ibmq_setup.ipynb     # Step-by-step access guide
│   ├── noise_mitigation/    # Techniques for real hardware noise
│   └── optimized_circuits/  # Circuits tuned for IBM's 5- and 7-qubit processors
└── 03_advanced_algorithms/  # Updated: Shor's, Grover's with hardware workarounds

Why This Matters for Learners

Simulators (like our existing qiskit_simulator_demo.ipynb) hide critical real-world challenges. With this update, you’ll learn to:

  • Optimize circuit depth to reduce decoherence (critical for IBM’s 100-200µs qubit coherence time)
  • Implement readout error mitigation using IBM’s calibration data
  • Compare results between ibmq_qasm_simulator (ideal) and ibmq_lima (physical hardware)

Hands-On Example: Grover’s Algorithm on IBM Hardware

Our most popular example — Grover’s search algorithm — now includes a hardware-optimized variant. We’ve reduced the number of CNOT gates by 40% and added dynamic qubit mapping to avoid IBM’s noisiest qubit connections.

# Key optimization in grover_hardware.ipynb
from qiskit.transpiler import PassManager
from qiskit.transpiler.passes import RemoveBarriers

# Reduce circuit depth for hardware
optimized_circuit = PassManager(RemoveBarriers()).run(grover_circuit)

# Map to IBM's qubit layout (avoids noisy connections)
optimized_circuit = transpile(
    optimized_circuit, 
    backend=ibmq_lima,
    initial_layout=[0, 1, 3]  #避开 IBM Lima's noisy qubit 2
)

Get Started Today

To run these examples on IBM hardware:

  1. Update your local repo: git pull origin main
  2. Follow docs/ibm_quantum_access.md to get your IBM API key
  3. Launch examples/02_ibm_quantum/ibm_setup.ipynb to test your connection
